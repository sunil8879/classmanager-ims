Creating a dump file in DBeaver for a Supabase project is the best way to back up your "Evergreen" logic and data. Because Supabase is built on PostgreSQL, DBeaver uses the native pg_dump utility to perform this task.

Here are the detailed steps to generate a high-quality .sql dump file:

Phase 1: Prerequisites

Before you start, make sure DBeaver is connected to your Supabase instance.

Go to your Supabase Dashboard > Project Settings > Database.

Find your Connection Info (Host, Port, Database name, User, and Password).

In DBeaver, ensure your connection works (Test Connection).

Note: Supabase requires SSL. In DBeaver connection settings, go to the SSL tab and check "Use SSL" if you haven't already.

Phase 2: Generating the Dump
1. Locate the Schema

In the DBeaver Database Navigator (left sidebar):

Expand your Supabase Connection.

Expand Databases.

Expand postgres.

Expand Schemas.

Right-click on public (this is where all your tables like students, attendance, etc., are stored).

2. Open the Dump Tool

Right-click public > Tools > Dump Database.

3. Select Objects

A window will appear showing the tables. Ensure all your tables are checked.

Click Next.

4. Configure the Dump Settings (Crucial for "Evergreen" Code)

This part is very important to ensure the file is clean and usable for other people.

Format: Choose Plain (This creates a readable .sql text file).

Filename Pattern: Choose where to save the file (e.g., C:\Backups\ClassManager_Dump.sql).

Encoding: Ensure it is set to UTF-8.

Dump Options (Check these boxes):

Insert commands (Best for moving data to new databases).

Include CREATE DATABASE statement (Uncheck this if you want to import it into an existing Supabase project).

Include DROP statement (Check this if you want the script to overwrite existing tables).

5. Set the PostgreSQL Binary (One-time Setup)

If this is your first time, DBeaver might ask for the "Local Client" path.

DBeaver needs the pg_dump.exe file on your computer to talk to Supabase.

If you don't have PostgreSQL installed locally, click "Download Client Binaries" when prompted. DBeaver will automatically download the necessary files for you.

6. Execute the Dump

Click Start.

A progress bar will appear. Once finished, DBeaver will say "Success."

Phase 3: Cleaning the Dump for Sale

When you sell your app globally for $1,999, you don't want the buyer to see your personal test data (like "Sameer Ghadi" or "Riya Vasarkar").

Open the .sql file in a text editor (Notepad++ or VS Code).

Look for INSERT INTO lines: These are your actual student records.

To sell the "Template" only: Delete the INSERT INTO lines but keep the CREATE TABLE and ALTER TABLE lines.

To sell with "Demo Data": Keep the INSERT lines but change the names to "John Doe" or "Test Student."
